    왜 draft도 하지않았는데 404 포스트로 나타나는걸까?

    April 19, 2021

    왜 아무리 써대도 이걸로는 잘 안되는걸까? 기껏해서 모든 글을 써놨지만, Book Theme은 계속해서 404 에러를 나타낸다.

    원인이 뭘까? #

    이걸 어떻게 해야 풀어낼 수 있을까? hugo serve -D 라는 명령어로는 localhost/1313에 나타나긴 한다마는… 왜 막상 hugo 로 해놨을 때는 아무것도 나타나지 않는걸까?

    알아낸 해결책 #

    1. git repo 에서 살펴보니, posts가 업데이트 되지 않고 있었다.
    2. 이유는 정확히 알 수 없었으나, docs 폴더 내에서 init 을 통해 push 하여 해결.

    시도했던 것들 #

    • config.yaml 에 다음과 같이 추가
    disablePathToLower: true
enableGitInfo: true

    허나 본래 되어있던 부분이라, 추가적으로 손을 대니 더 엉망이 되서 다시 추가함.

    • 온갖 hugo -{} 태그들 --minify, -t book, -D - 허나 위 방법들과는 전혀 관련이 없었음

    어떻게 했어야 시간을 아끼면서 빠르게 해결책을 찾을 수 있었을까? #

    글쎄… 여러가지 docs를 구글링해가며 찾아봤으나, 뾰족한 방법은 없었음.. 일단 구글링부터 해내는게 아니라, 어디서 무엇이 문제였는지 파악했었어야할까? 가장 확률이 높아보이는 가정들을 리스팅 한 뒤, 그 중에서 가장 문제로 보이는 것들부터 하나씩 소거 해나갔어야했을까?

    가장 문제가 있어보이는 것들부터 했었어야하는데, 미처 리스팅 없이 구글링부터 해낸게 여러 문제를 일으키지 않았나 싶다. 빠르게 인스턴트하게 해결책을 찾아내는 방법은 어찌보면 구글링이기도 하니까.

    다짜고짜 구글링을 해내는건 하지만 다른 여러 진정한 해결책을 찾거나, 문제들을 풀어가게 만드는 일에 있어서는 가장 좋지 않은 방법일까?
